<p>Twitter bietet über die Einbindung des Twitter Buttons einige nette Funktionalitäten – jeder Nutzer sieht sofort wie oft die Seite schon getwittert wurde und der Nutzer kann es mein einem Klick ebenfalls machen. Das Ganze hat natürlich seinen Preis:</p> <p>- Durch das Einbinden des Twitter Javascripts ist die Sache mit dem Datenschutz komplizierter.</p> <p>- Man benötigt das Twitter Javascript</p> <p>- “Hintergrund” Prozesse können so nicht auf diese “Tweet”-Zahl zugreifen</p> <h3>Twitters Steiniger Weg über die offizielle API</h3> <p>In den Development-Foren von Twitter scheint dies eine beliebte Frage zu sein: Wie kommt man an die Zahl ran die der Twitter Button darstellt. Offiziell legt Twitter Wert darauf das man die Streaming API verwendet. Hier findet sich sogar ein <a href="https://dev.twitter.com/discussions/5653">Beispiel</a>. Allerdings ist diese API nur mit einem authentifizierten Dev-Account möglich. Wer viele Seite hat, dem könnten die diversen Beschränkungen von Twitter ein Strich durch die Rechnung machen.</p> <h3>Der einfache Weg über http://urls.api.twitter.com/1/urls/count.json?url=…</h3> <p>Tja – es ist nicht schwer nachzuvollziehen woher der magische Button die Daten bekommt. Auch wenn Twitter <a href="https://dev.twitter.com/discussions/5839">mehrfach darauf hinweisst</a>, dass dies keine offizielle API ist, ist die Verwendung ziemlich einfach:</p> <p>GET Request: <a title="http://urls.api.twitter.com/1/urls/count.json?url=http://blog.codeinside.eu/2014/04/15/source-code-verffentlichen-aber-bitte-mit-lizenz/" href="http://urls.api.twitter.com/1/urls/count.json?url=http://blog.codeinside.eu/2014/04/15/source-code-verffentlichen-aber-bitte-mit-lizenz/">http://urls.api.twitter.com/1/urls/count.json?url=http://blog.codeinside.eu/2014/04/15/source-code-verffentlichen-aber-bitte-mit-lizenz/</a></p> <p>Ergebnis: </p><pre>{"count":7,"url":"http:\/\/blog.codeinside.eu\/2014\/04\/15\/source-code-verffentlichen-aber-bitte-mit-lizenz\/"}</pre>
